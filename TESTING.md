# Как убедиться, что запущена актуальная версия

## 0. Почему раньше «ничего не менялось»

В `electron/main/index.ts` загрузка Vite в dev была привязана к `NODE_ENV === 'development'`. В Electron main этот флаг часто **не** равен `development`, поэтому окно всегда открывало **старый** `out/renderer/index.html` вместо dev-сервера — правки в `src/` не попадали в окно.

Сейчас dev-сервер берётся, если задан `ELECTRON_RENDERER_URL` и приложение **не** упаковано (`!app.isPackaged`).

При `npm run dev` в **терминале** должна появиться строка:
`[PreviewV] Loading renderer from Vite dev server: http://...`

Если её **нет** — открывается файловый бандл из `out/`, а не Vite.

## 1. Разработка (из исходников)

В корне проекта (папка с `package.json`):

```bash
cd C:\Users\nfamf\Desktop\PreviewV
npm install
npm run dev
```

В **правом нижнем углу** должна быть строка вида:

- **`DEV · pv-2025-03-19c`** — зелёный `DEV` значит открыт режим Vite + hot reload.

Если там **`PROD`** или **нет** актуальной метки `pv-…` — вы смотрите **не тот** бандл (старый exe или другая копия проекта).

## 2. Установленный .exe (release)

Изменения в коде **не попадают** в уже собранный `PreviewV` в `release/`, пока вы снова не соберёте:

```bash
npm run build:win
```

После установки/запуска нового exe в углу будет **`PROD`** (не `DEV`) — это нормально. Метка **`pv-…`** должна совпадать с текущей в `src/components/ViewportHud.tsx` (`RENDERER_BUILD_ID`).

## 3. Быстрая проверка функций

| Действие | Ожидание |
|----------|----------|
| Кнопка **Fit all (A)** или клавиша **A** (клик по холсту сначала) | Все плитки в кадре |
| Видео-плитка | Снизу полоска: Play, слайдер, время |
| `npm run dev` | В углу **`DEV`** |

## 4. Если «ничего не меняется»

1. Закройте все окна PreviewV.
2. Убедитесь, что запускаете **`npm run dev`** из **этой** папки репозитория.
3. Удалите кэш сборки и пересоберите: `npm run clean` затем `npm run dev` (скрипт `clean` в `package.json`).
4. Не открывайте старый ярлык на exe из `release`, если хотите видеть последние правки без `build:win`.
