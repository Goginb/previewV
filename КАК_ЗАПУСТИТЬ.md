# Как правильно открыть PreviewV

## Вариант A — разработка (свежий код из папки проекта)

1. Открой **папку проекта**, где лежит `package.json` (например `C:\Users\...\Desktop\PreviewV`).
2. В терминале (PowerShell или cmd):

```powershell
cd путь\к\PreviewV
npm install
npm run dev
```

3. Должно открыться окно приложения.
4. В **терминале** при старте должна появиться строка вида:

   `[PreviewV] Loading renderer from Vite dev server: http://127.0.0.1:...`

   Если этой строки **нет** — окно может грузить **старый** собранный интерфейс из `out\`, а не актуальный `src`. Тогда:

```powershell
npm run clean
npm run dev
```

5. Внизу справа в окне должна быть метка **`DEV`** (режим разработки).

---

## Вариант B — установленная программа (.exe после сборки)

1. Собери проект:

```powershell
cd путь\к\PreviewV
npm install
npm run build:win
```

2. Готовые файлы обычно в папке **`release\`** (установщик и/или portable `.exe`).
3. Запускай **тот exe**, который только что собрал — старый ярлык на рабочем столе может вести на **прошлую** версию.

В собранном приложении в углу будет **`PROD`**, не **`DEV`** — это нормально.

---

## Что не так, если «ничего не меняется»

| Проблема | Что сделать |
|----------|-------------|
| Запускаешь **старый** `.exe` из `release` | Снова `npm run build:win` и открой **новый** файл из `release` |
| `npm run dev` в **другой** копии проекта | Убедись, что `cd` в ту же папку, где лежит код |
| Нет строки `Loading renderer from Vite` | `npm run clean` + `npm run dev` |

---

## Горячие клавиши (напоминание)

Кликни по **холсту** (тёмная область), чтобы окно было в фокусе, затем:

- **A** — показать все плитки в кадре  
- **Пробел + ЛКМ** — панорама (если настроено)

---

## Минимальная проверка, что всё открылось правильно

1. `npm run dev`  
2. В терминале есть **`Loading renderer from Vite dev server`**  
3. В окне справа снизу **`DEV`**  

Если все три пункта выполняются — запущена правильная версия.
