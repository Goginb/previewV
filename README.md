# PreviewV

Настольное приложение (**Electron**): бесконечный холст для **референса по видео** — плитки с роликами, кадрами, заметками и простым рисованием по изображениям. Проекты сохраняются в файлы **`.previewv`** (JSON).

## Запуск в режиме разработки

1. Клонируйте репозиторий и перейдите в папку проекта (где лежит `package.json`).
2. В терминале (PowerShell или cmd):

```powershell
cd путь\к\PreviewV
npm install
npm run dev
```

3. Должно открыться окно приложения.
4. В **терминале** при старте желательно увидеть строку вида:

   `[PreviewV] Loading renderer from Vite dev server: http://127.0.0.1:...`

   Если её **нет**, интерфейс может подгружаться из **старой** сборки в `out\`, а не из актуального `src\`. Тогда:

```powershell
npm run clean
npm run dev
```

5. В окне справа снизу должна быть метка **`DEV`** — это режим разработки.

## Сборка установщика (Windows)

### Установщик NSIS (рекомендуется для пользователей)

Собирает **полноценный мастер установки** (`PreviewV Setup <версия>.exe` в папке **`release\`**):

- пошаговый мастер (не «one-click»);
- можно **указать папку установки** («Изменить» / выбор каталога);
- при необходимости запрос **повышения прав** (установка, например, в `Program Files`);
- после установки: ярлык **PreviewV** в меню «Пуск», ярлык на **рабочем столе** (если не снять галочки в мастере), пункт **«Удаление программ»**;
- опция **запустить приложение после завершения** (`runAfterFinish`).

Обычный ярлык/запись в меню относятся к **установленной** копии в выбранной папке, а не к одному exe «вручную».

```powershell
cd путь\к\PreviewV
npm install
npm run build:win
```

Результат: **`release\PreviewV Setup 0.2.0.exe`** (номер версии берётся из `package.json`).

### Portable (без установки)

Один файл `.exe`, без мастера — **ярлыки в системе не создаются** (удобно для флешки или теста):

```powershell
npm run build:win:portable
```

### Оба артефакта сразу

```powershell
npm run build:win:all
```

---

Запускайте **свежесобранный** установщик или portable — старый ярлык может указывать на прошлую версию.

В собранном приложении в углу будет **`PROD`**, не **`DEV`** — это нормально.

## Проекты `.previewv`

| Действие | Как |
|----------|-----|
| Формат | JSON, расширение **`.previewv`** — плитки, масштаб и смещение холста (viewport). |
| Открыть | **Файл → Открыть…** или **Ctrl+O**; недавние — в меню **Файл**. |
| Добавить папку | **Файл → Add folder…** — все подходящие видео и картинки из папки **рекурсивно** (вложенные папки тоже). Если файл уже есть на холсте, спросит: добавить копии ещё раз, пропустить дубликаты или отменить. |
| Сохранить | **Ctrl+S** |
| Сохранить как | **Ctrl+Shift+S** |
| Закрыть проект (без выхода из программы) | **Файл → Закрыть проект** или **Ctrl+W** |
| Несохранённые изменения | При закрытии окна, смене проекта или открытии другого файла будет запрос: сохранить / не сохранять / отмена. |
| Имя в интерфейсе | Вверху окна и в заголовке отображается текущий файл; точка **•** — есть несохранённые правки. |

Ассоциация файлов **`.previewv`** с приложением настраивается при установке сборки (см. `package.json` → `build.fileAssociations`).

## Холст: видео, кадры, заметки

- **Видео:** перетащите файлы (например `.mp4`) на холст. Локальные файлы открываются через схему `media://` с поддержкой **byte-range** — для MP4 корректно работает перемотка и таймлайн.
- **Кадр из ролика:** выделите плитку видео, нажмите **F3** — рядом появится плитка-изображение.
- **Заметки:** **Ctrl+N** — новая заметка у курсора.
- **Рисование по изображению:** выделите плитку-картинку, **F4** — режим редактирования (плитка расширяется); **F4** снова или **Done** — выход. В режиме рисования отмена штриха — **Ctrl+Z** (в контексте плитки).
- **Изображения EXR/DPX** и др.: при перетаскивании нужен локальный путь к файлу; декодирование через **ffmpeg** (см. зависимости проекта).
- **Окно поверх других:** **Alt+Shift+A** — включить, **Alt+Shift+B** — выключить; то же в меню **View** (и чекбокс «Always on top»). Вверху окна при включении показывается метка **Pinned**.
- **Панорама:** зажмите **Пробел** и тяните ЛКМ (если включено в вашей сборке).
- **Масштаб:** колесо мыши с **Ctrl** (или тачпад).
- **Выделение рамкой:** протяните мышь по пустому месту; **Shift+клик** — добавить к выделению.
- **Вписать всё в кадр:** клавиша **A** (без модификаторов) или кнопка **Fit all**.
- **Сетка медиа:** **L** — все **видео** на холсте выстраиваются в ряды по **20** штук, затем отдельным блоком ниже — все **картинки** (тоже по 20 в ряд); плитки не пересекаются.
- **Сетка всего холста:** **\\** (обратная косая) или кнопка **Grid pack** — **все** плитки (видео, картинки, заметки) в одну сетку по порядку сверху-вниз, до **20** в ряд, без пересечений (якорь — левый верхний угол бывшего «облака» плиток).
- В правом нижнем углу: **Stop all / Play all** — глобальная пауза автовоспроизведения видео на холсте.

## Горячие клавиши

| Комбинация | Действие |
|------------|----------|
| **Ctrl+O** | Открыть проект |
| **Ctrl+S** | Сохранить |
| **Ctrl+Shift+S** | Сохранить как… |
| **Ctrl+W** | Закрыть проект |
| **Ctrl+A** | Выделить все плитки на холсте (в поле текста заметки — выделить текст) |
| **A** | Подогнать все плитки в видимую область |
| **Ctrl+Z** / **Ctrl+Shift+Z** | Отмена / повтор |
| **Ctrl+C** / **Ctrl+V** | Копировать / вставить плитки |
| **Delete** | Удалить выделенное |
| **Shift+D** | Дублировать выделение |
| **L** | Упорядочить **все** видео и **все** картинки: видео — ряды по 20, картинки — отдельно ниже, ряды по 20 |
| **\\** | **Grid pack:** все плитки (видео, картинки, заметки) в сетку без наложений, до 20 в ряд |
| **Ctrl+N** | Новая заметка |
| **F3** | Снять кадр с выбранного видео |
| **F4** | Режим рисования по выбранной плитке-изображению (вкл./выкл.) |
| **Alt+Shift+A** | Закрепить окно поверх других (always on top) |
| **Alt+Shift+B** | Снять закрепление окна |
| **F1** | Справка в окне приложения (**Help → User guide**) |

*На macOS вместо **Ctrl** часто используется **⌘**; **Alt** ≈ **Option**.*

## Если «ничего не меняется» после правок

| Проблема | Что сделать |
|----------|-------------|
| Запускается **старый** `.exe` из `release` | Снова `npm run build:win` и откройте **новый** файл из `release` |
| `npm run dev` из **другой** копии папки проекта | Убедитесь, что `cd` в ту же папку, где лежит актуальный код |
| Нет строки `Loading renderer from Vite` | `npm run clean`, затем `npm run dev` |
